PMON/2000
On-line Documentation

The PMON/2000 BootROM Monitor (PMON/2000) permits full-featured debugging to be performed on a target board that has a serial port, 512KB of ROM, and 128KB of RAM that can be dedicated to the Prom Monitor software.

A PMON/2000 system consists of two components: A Host, and a Target, as shown below.

For information on configuring and building the PMON/2000 debugger, please refer to the section on Getting Started.

PMON/2000 Features

  • A Powerful set of Assembly-level debugging commands
  • Supports Source-Level Debuggers
  • Source is distributed license-free
  • Unix-like user Interface (including command history/editing)
  • Display/set CPU Registers
  • Display/ set Memory
  • Disassemble Memory
  • Search/ Copy/ Fill Memory
  • Multiple Breakpoints
  • Single Step Execution (including PROM)
  • Downloadable Symbols for S-records and Fast-format records
  • Download via RS232 or ETHERNET
  • Supports Flash Memory for Non-Volatile Settings
  • Limitations.
  • Frequently Asked Questions
